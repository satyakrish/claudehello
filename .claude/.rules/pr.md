# PR Rules

- Every PR must close exactly one issue (`Closes #N` in the body)
- Every PR opened under AI-SDLC must be labelled `ai-sdlc`
- PR title format: `<type>: <short description>` (e.g. `feat: add hello world page`)
- PR body must include: What changed, Why, How to test
- CI must be green before requesting review
- Do not merge without CI passing
- Do not bundle multiple issues into one PR
- Keep PRs small and focused — if it's getting large, split it
