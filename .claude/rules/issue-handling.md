# Issue Handling Rules

- Only implement issues labelled `ai-sdlc` — stop immediately if the label is absent
- Understand the intent and goal of the issue, not just the technical surface
- Identify acceptance criteria, in-scope items, and explicitly list what is out of scope before starting
- If acceptance criteria are missing or ambiguous, state the assumption clearly in the PR body and continue
- Never implement anything outside the issue scope — flag it in the PR body under "Out of Scope / Follow-ups"
- Create a new issue for out-of-scope work only when the human explicitly asks — never proactively
